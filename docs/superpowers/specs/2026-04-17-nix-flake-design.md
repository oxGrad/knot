# Nix Flake Design

**Date:** 2026-04-17
**Status:** Approved

## Overview

Add a `flake.nix` that provides a reproducible dev shell, a `nix build` target for the `knot` binary, and distro-isolated integration testing (Ubuntu/Fedora/Arch) as a Nix-native alternative to the existing Docker workflow. The Dockerfile and docker-compose.yml are left intact; Nix is an additive parallel path.

## Goals

- `nix develop` gives a complete Go toolchain + linters without manual installs
- `nix build` produces the `knot` binary
- `nix run .#test-ubuntu/fedora/arch` replicates `make test-ubuntu/fedora/arch` using real distro base images
- All image base digests are pinned in the flake for reproducibility

## Non-goals

- Replacing the existing Dockerfile / docker-compose.yml
- NixOS-only VM tests (requires real Ubuntu/Fedora/Arch bases)
- CI migration (flake is local-dev only for now)

---

## Section 1: `packages.default` ‚Äî `nix build`

Uses `pkgs.buildGoModule` to compile the `knot` binary:

- Go version read from `go.mod` (currently 1.26.2)
- `vendorHash` derived from `go.sum`
- Cross-compiled for linux/amd64 via `pkgs.pkgsCross.gnu64` so the binary is correct for Linux containers when building from macOS
- Output: `result/bin/knot`

```
nix build          # builds ./result/bin/knot
```

## Section 2: `devShells.default` ‚Äî `nix develop`

Provides the full local development toolchain:

| Tool | Purpose |
|------|---------|
| `go` | Pinned to match `go.mod` |
| `golangci-lint` | Linter (matches CI) |
| `goreleaser` | Release tooling |
| `git` | Required by goreleaser changelog |

Shell sets `GOPATH` and `GOMODCACHE` to `.nix-go/` inside the project root so module cache is isolated from the system Go installation.

```
nix develop        # enter dev shell
go test ./...      # works immediately
golangci-lint run  # works immediately
```

## Section 3: OCI images ‚Äî `nix build .#images.<distro>`

Each of the three distro images is built in two steps:

### Step 1 ‚Äî `dockerTools.pullImage`

Fetches the real distro base image pinned by SHA256:

| Image | Tag | Pinned |
|-------|-----|--------|
| `ubuntu` | `24.04` | sha256 in flake |
| `fedora` | `40` | sha256 in flake |
| `archlinux` | `latest` | sha256 in flake |

Pinning ensures `nix build` is reproducible across machines and time.

### Step 2 ‚Äî `dockerTools.buildLayeredImage`

Layers on top of the pinned base:

- Copies the linux/amd64 `knot` binary to `/usr/local/bin/knot`
- User: `oxGrad` (uid 1001), matching the Dockerfile
- Working dir: `/home/oxGrad`
- Default cmd: `["zsh"]`

Output is an OCI tarball at `result`. Three packages are exposed:

```
nix build .#images.ubuntu   # ‚Üí result (OCI tarball)
nix build .#images.fedora
nix build .#images.arch
```

## Section 4: Test apps ‚Äî `nix run .#test-<distro>`

Each app is a bash script (written inline in the flake) that chains:

1. `nix build .#images.<distro>` ‚Äî builds the OCI tarball
2. `docker load < result` ‚Äî loads into the local Docker daemon
3. `docker run --rm -it knot-<distro>` ‚Äî interactive shell in the container

These are direct replacements for `make test-ubuntu/fedora/arch`.

```
nix run .#test-ubuntu    # replaces: make test-ubuntu
nix run .#test-fedora    # replaces: make test-fedora
nix run .#test-arch      # replaces: make test-arch
```

### Makefile additions

Three new targets are added alongside (not replacing) the existing ones:

```makefile
nix-test-ubuntu:
    nix run .#test-ubuntu

nix-test-fedora:
    nix run .#test-fedora

nix-test-arch:
    nix run .#test-arch

nix-test-all:
    nix run .#test-ubuntu -- knot version
    nix run .#test-fedora -- knot version
    nix run .#test-arch   -- knot version
```

```

Makefile                   # update ‚Äî add nix-test-* targets
```

## Flake structure (pseudocode)

Outputs are generated for both `x86_64-linux` and `aarch64-darwin` (macOS) using a `forAllSystems` helper. The binary and OCI images are always cross-compiled to linux/amd64. Dev shell and apps work on both platforms.

```nix
{
  inputs.nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";

  outputs = { nixpkgs, ... }:
    let
      forAllSystems = nixpkgs.lib.genAttrs [ "x86_64-linux" "aarch64-darwin" ];
    in {
      packages = forAllSystems (system: let
        pkgs = nixpkgs.legacyPackages.${system};
        crossPkgs = pkgs.pkgsCross.gnu64;  # always linux/amd64
      in {
        default = crossPkgs.buildGoModule { ... };  # knot binary
        "images/ubuntu" = pkgs.dockerTools.buildLayeredImage { ... };
        "images/fedora" = ...;
        "images/arch"   = ...;
      });

      devShells = forAllSystems (system: let
        pkgs = nixpkgs.legacyPackages.${system};
      in {
        default = pkgs.mkShell {
          buildInputs = [ pkgs.go pkgs.golangci-lint pkgs.goreleaser pkgs.git ];
        };
      });

      apps = forAllSystems (system: {
        # Each script: nix build image ‚Üí docker load ‚Üí docker run
        # Trailing args after `--` are forwarded to `docker run`
        test-ubuntu = { type = "app"; program = "<script>"; };
        test-fedora = ...;
        test-arch   = ...;
      });
    };
}
```

## Open questions resolved during design

- **Cross-compilation:** macOS users build linux/amd64 binary via `pkgsCross.gnu64`; the flake exposes outputs for `x86_64-linux` and `aarch64-darwin` systems
- **Base image pinning:** SHA256 digests fetched once via `nix flake update` and committed to `flake.lock`; updating pins is explicit (`nix flake update`)
- **Docker dependency:** `nix run .#test-*` still requires Docker to be installed locally for `docker load` and `docker run`; Nix owns the build, Docker owns the runtime
